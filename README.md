# clowk-terminal

Clowk color schemes for your terminal — **one palette, many formats.**

A small [Bun](https://bun.sh) + TypeScript builder that takes a single canonical
palette (sourced from the [`clowk-vscode-theme`](https://github.com/thadeu/clowk-vscode-theme)
Cursor/VS Code theme) and generates ready-to-install color schemes for every
terminal that matters.

## Supported terminals

| Terminal | File | Install |
|---|---|---|
| **Ghostty** | `ghostty/clowk-night` | drop in `themes/` + `theme = clowk-night` |
| **cmux** | _use the Ghostty theme_ | cmux renders through Ghostty |
| **iTerm2** | `iterm2/Clowk Night.itermcolors` | import as a Color Preset |
| **macOS Terminal.app** | `terminal-app/Clowk Night.terminal` | double-click to import |
| **Alacritty** | `alacritty/clowk-night.toml` | `general.import` |
| **kitty** | `kitty/clowk-night.conf` | `include` |
| **WezTerm** | `wezterm/clowk-night.toml` | `colors/` + `color_scheme` |
| **Windows Terminal** | `windows-terminal/clowk-night.json` | paste into `schemes` |

## Get the themes

**Download** from the latest [release](https://github.com/thadeu/clowk-terminal/releases).
Each terminal has its own file, named `<terminal>-clowk-night.*` — so you grab only
what you need:

| Asset | For |
|---|---|
| `clowk-terminal-themes.zip` | everything, in one download |
| `ghostty-clowk-night` | Ghostty / cmux |
| `iterm2-Clowk-Night.itermcolors` | iTerm2 |
| `terminal-app-Clowk-Night.terminal` | macOS Terminal.app |
| `alacritty-clowk-night.toml` | Alacritty |
| `kitty-clowk-night.conf` | kitty |
| `wezterm-clowk-night.toml` | WezTerm |
| `windows-terminal-clowk-night.json` | Windows Terminal |

The install steps below use the `dist/…` paths from a local build; if you downloaded
a release asset, just point at the downloaded file instead.

**Or build them yourself:**

```bash
bun install            # only needed for lint / typecheck
bun run build          # render every format into dist/
bun run verify         # (macOS) lint plists + round-trip the Terminal.app archive
```

`dist/` is git-ignored — it's a build artifact, produced on demand and attached
to GitHub releases.

## Install per terminal

### Ghostty (and cmux)

```bash
mkdir -p ~/.config/ghostty/themes
cp "dist/ghostty/clowk-night" ~/.config/ghostty/themes/clowk-night
```

Then in `~/.config/ghostty/config`:

```
theme = clowk-night
```

> **cmux** reads your Ghostty config for colors, so the Ghostty theme *is* the cmux
> theme. Heads up: switching cmux's Light/Dark mode rewrites the Ghostty theme and
> overrides custom colors ([cmux#3523](https://github.com/manaflow-ai/cmux/issues/3523))
> — pin a fixed mode if you want clowk to stick.

### iTerm2

`Settings → Profiles → Colors → Color Presets… → Import…`, pick
`Clowk Night.itermcolors`, then select **Clowk Night** from the same menu.

### macOS Terminal.app

Double-click `Clowk Night.terminal` (or `open "Clowk Night.terminal"`). Then
`Settings → Profiles → Clowk Night → Default ⌄` to make it the default profile.

### Alacritty

```bash
cp dist/alacritty/clowk-night.toml ~/.config/alacritty/
```

In `alacritty.toml`:

```toml
[general]
import = ["~/.config/alacritty/clowk-night.toml"]
```

### kitty

```bash
cp dist/kitty/clowk-night.conf ~/.config/kitty/
```

In `kitty.conf`:

```
include clowk-night.conf
```

### WezTerm

```bash
mkdir -p ~/.config/wezterm/colors
cp dist/wezterm/clowk-night.toml ~/.config/wezterm/colors/
```

In `wezterm.lua`:

```lua
config.color_scheme = "Clowk Night"
```

### Windows Terminal

Open Settings → `Open JSON file`, and paste the object from
`windows-terminal/clowk-night.json` into the `"schemes"` array. Then set
`"colorScheme": "Clowk Night"` on a profile.

## How it works

```
palettes/*.json  ──►  src/generators/*  ──►  dist/<format>/<scheme>.<ext>
 (source of truth)     (one per format)        (build artifact)
```

- **`palettes/clowk-night.json`** — the canonical palette: 16 ANSI colors plus
  background / foreground / cursor / selection. Generated by
  `bun run import`, which reads the `terminal.ansi*` keys from the VS Code theme.
- **`src/generators/<id>.ts`** — each exports a `Generator`
  (`{ id, dir, ext, filename, render }`). `render(palette)` returns the file body.
- **`src/build.ts`** — runs every generator over every palette.

### Add a terminal format

Create `src/generators/foo.ts` exporting a `Generator`, then register it in
`src/generators/index.ts`. Done — it runs against every palette.

### Add a scheme (e.g. a light variant)

Drop another `palettes/<slug>.json` with the same shape. Every format gets it
for free.

## Scripts

| Command | What it does |
|---|---|
| `bun run import` | regenerate `palettes/clowk-night.json` from `clowk-vscode-theme` |
| `bun run build` | render every format into `dist/` |
| `bun run verify` | (macOS) lint the plists + round-trip the Terminal.app archive |
| `bun run all` | import + build + verify |
| `bun run typecheck` | `tsc --noEmit` |
| `bun run lint` | ESLint |
